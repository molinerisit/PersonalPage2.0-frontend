Venta Simple — SaaS de gestión para comercios

Problema

Los comercios chicos manejaban ventas con cuaderno o planillas: caja lenta, errores de cálculo, stock sin control y sin visibilidad cuando el dueño no estaba en el local.

Solución

  • POS de escritorio (JavaFX) que funciona offline y sincroniza solo.
  • Panel web (React) con métricas, catálogo y multi-empresa.
  • App móvil para escanear e inventariar desde el celular.
  • Facturación AFIP/ARCA integrada y cumplimiento fiscal.
  • IA aplicada al negocio: analiza datos de ventas y stock para sugerir mejoras y decisiones.

Mi rol

Diseño y desarrollo end-to-end: arquitectura, backend, frontend, app móvil, integración fiscal, despliegue (Vercel/Railway) y soporte a clientes en producción.

Capturas reales del producto

Arquitectura

  • POS desktop (JavaFX + Java): base local H2/SQLite, cola de sincronización y reintentos, impresión térmica y lector de barras.
  • Backend (Java + Spring Boot): API REST, autenticación por roles con Spring Security (JWT), Hibernate/JPA, multi-empresa, integración con AFIP/ARCA.
  • Panel web (React + Tailwind): KPIs, catálogo online, comprobantes y onboarding.
  • App móvil: scanner, inventario y carrito conectados a la misma API.
  • Datos & deploy: PostgreSQL, Docker, Vercel (web), Railway (backend).
  • Integraciones: Mercado Pago (cobros + QR), Meta / WhatsApp Business API y Brave Search API (búsqueda de imágenes de productos).

Desafíos técnicos resueltos

  • Sincronización offline-first con resolución de conflictos entre POS y nube.
  • Integración real con facturación electrónica AFIP/ARCA (certificados, CAE, multi-empresa).
  • Idempotencia en operaciones de venta/stock para evitar duplicados.
  • Empaquetado y releases del POS de escritorio (JavaFX) con Maven vía CI (GitHub Actions).
